# 开发指南

## 一、项目概述

运城果业记账库存管理系统是一个专为果商设计的手机管理系统，解决记账乱、库存不清、欠款对不上、客户管理难等问题。

### 技术栈

- **后端**：Spring Boot 2.7.18 + MyBatis Plus 3.5.5 + MySQL 8.0 + Redis 7.x
- **前端**：微信小程序原生开发
- **JDK版本**：JDK 1.8
- **构建工具**：Maven 3.6+

## 二、后端启动步骤

### 前置要求

- JDK 1.8+
- Maven 3.6+
- MySQL 8.0+
- Redis 7.x+（可选，初期可注释Redis配置）

### 1. 创建数据库

```bash
# 登录MySQL
mysql -u root -p

# 执行初始化脚本
source d:/user/workspace_fruit/docker/mysql/init/schema_new.sql
```

### 2. 修改配置文件

编辑 `fruit-admin/src/main/resources/application.yml`，修改数据库和Redis连接信息：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/fruit_management
    username: root
    password: 你的密码
  data:
    redis:
      host: localhost
      port: 6379
```

**注意**：如果没有Redis，可以先注释掉Redis相关配置和依赖。

### 3. 启动后端服务

```bash
cd fruit-admin
mvn spring-boot:run
```

### 4. 访问API文档

启动成功后，访问：

```
http://localhost:8080/doc.html
```

### 5. 测试登录接口

在Knife4j文档页面测试：

- 接口路径：`POST /api/auth/login`
- 请求参数：

```json
{
  "username": "admin",
  "password": "123456"
}
```

## 七、如何生成BCrypt密码

如果需要创建新用户，可以使用项目中的 `PasswordGenerator.java` 工具类，或使用以下Java代码生成BCrypt密码哈希：

```java
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;

public class PasswordGenerator {
    public static void main(String[] args) {
        BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();
        String rawPassword = "你的密码";
        String encodedPassword = encoder.encode(rawPassword);
        System.out.println("加密后的密码：" + encodedPassword);
    }
}
```

## 八、修改密码功能说明

### 接口信息

- **路径**：`POST /api/auth/change-password`
- **认证**：需要Bearer Token
- **请求参数**：

```json
{
  "oldPassword": "旧密码",
  "newPassword": "新密码"
}
```

### 验证规则

1. 旧密码必须正确
2. 新密码长度不能少于6位
3. 修改成功后需要重新登录

## 九、常见问题

### 1. 启动报错：Invalid value type for attribute 'factoryBeanObjectType'

**原因**：Spring Boot版本与MyBatis Plus版本不兼容
**解决**：已降级到Spring Boot 2.7.18 + MyBatis Plus 3.5.5

### 2. 启动报错：Failed to start bean 'documentationPluginsBootstrapper'

**原因**：Spring Boot 2.6+默认路径匹配策略与Swagger不兼容
**解决**：已在application.yml中添加配置：

```yaml
spring:
  mvc:
    pathmatch:
      matching-strategy: ant_path_matcher
```

### 3. 登录提示密码错误

**原因**：数据库中的密码哈希值不正确
**解决**：使用schema\_new\.sql重新初始化数据库，或使用PasswordGenerator生成正确的密码哈希

### 4. 编译错误：不兼容的类型

**原因**：JDK版本问题
**解决**：确保使用JDK 1.8，已在pom.xml中配置
